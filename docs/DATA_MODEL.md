# 数据模型与隐私边界

首版 PostgreSQL 结构位于 `db/migrations/001_initial.sql`。

## 记录内容

- 用户、网关密钥 ID 与密钥哈希；
- 上游及加密后的凭证；
- 用户请求状态、时间、模型、Token 与金额；
- 每次上游尝试的凭证指纹、状态、耗时和成本；
- 请求与尝试之间的一对多关系。
- 邮箱账号会话、个人资料和余额流水；

## 禁止记录

- 网关请求日志中的用户提示词和模型输出正文；
- 用户网关密钥明文；
- 上游密钥明文；
- 用户本地 Codex 配置内容；
- 用户聊天提示词、模型回复和本地会话标题。

金额统一存储为整数微单位，避免浮点误差。真实凭证必须由应用层使用 KMS 或独立主密钥加密后写入 `encrypted_secret`。

从 `v1.1` 开始，会话与消息只保存在客户端安装目录的 `data/conversations` 中。迁移 `011_local_conversations_only.sql` 会删除服务器原有的聊天会话表及其内容；兼容旧客户端的 `/sync/*` 路由只返回临时响应，不写入数据库。

用户密码只保存经过带随机盐的 scrypt 派生验证值，不保存、记录或回传明文密码。连续失败会在账户记录中触发临时锁定；验证码仍用于新用户注册和密码重置。

## 登录与数据处理说明

- Windows 登录会话令牌使用 Windows 数据保护机制加密，密文只写入客户端所选安装目录的 `data/session.dat`；服务端保存会话摘要与有效期，不保存客户端令牌明文。
- 邮箱地址、网名和头像属于账号资料，会保存在服务端，以便 Windows 与 Android 使用同一账号。密码明文不会保存。
- 对话请求会经过泺栋 Chat 网关转发给上游模型服务，因此网关会在处理请求时接触本次提示词和模型输出，但不会将正文写入请求日志或数据库。
- 服务端只记录请求时间、状态、模型、Token 用量、费用、上游凭证指纹和耗时等运行数据，用于计费、故障切换、安全审计与服务告警。
- 对话正文、回复内容、会话标题和客户端历史记录仅保存在用户设备中。用户退出账号时，服务端会使对应会话失效。

本说明最近更新于 2026-07-15。如数据处理范围发生变化，应同步更新本文件与官网隐私入口。
